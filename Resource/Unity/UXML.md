# 정의

XML에서 차용한 Unity eXtensible Markup Language이라는 뜻이다.
말 그대로 XML과 비슷한 방식으로 작동하며 **UI 요소의 계층적 구조와 레이아웃을 선언적으로 정의**한다.

- **CSS와 유사한 용도로 USS(Unity Style Sheet)를 사용해서 시각적인 스타일을 적용**할 수 있다. 이를 통해 구조(UXML)와 스타일(USS)을 분리하여 관리하여 개발의 유연성과 유지보수성을 높일 수 있다.

- - **주로 Unity 에디터 내에서 커스텀 에디터의 UI를 디자인하고 개발**하는 데 사용되며, **런타임(게임 실행 중) UI를 구축하는 데도 활용할 수도 있다.** (단, 런타임 UI는 `UnityEngine.UIElements` 네임스페이스의 요소를 사용하며, 에디터 전용 요소는 제외해야 동작함)

- Unity 내에서 제공하는 **UI 빌더(UI Builder)를 통해 작성할 수 있어 생산성을 크게 높일 수 있다.** UI 빌더는 UXML 파일의 내용을 실시간으로 미리보기하고, 속성과 스타일을 편집하는 기능을 제공한다.
